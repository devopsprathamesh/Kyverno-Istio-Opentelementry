# Planned Component Version Matrix

**Version research performed:** 2026-07-23
**Researcher:** Automated research pass against official project documentation and release pages (Kubernetes, Cilium, Istio, Kyverno, OpenTelemetry, Prometheus, Grafana, Jaeger, Loki, HashiCorp, Oracle, Canonical, containerd, Helm), cross-checked with `endoflife.date` and GitHub release listings where official pages did not surface an exact patch tag.

This document records **planned** versions only. No component listed here has been installed or tested in this repository yet — see [`VALIDATION-STATUS.md`](VALIDATION-STATUS.md) for what has actually been verified. All versions are pinned to exact releases; floating `latest` tags are prohibited (see [ADR-010](DECISIONS.md#adr-010-version-pinning)). Re-verify every "Pinning strategy: re-check at install time" row against the linked source before actually provisioning, since patch releases ship continuously.

## Cross-cutting compatibility decision

Kubernetes 1.36.2 is the newest upstream release at research time, but **Kubernetes 1.35.6 is the planned base version**, because it is the newest minor release that falls inside the *tested* compatibility window of all three cluster-attached tools this repository installs:

| Tool | Tested Kubernetes range (at research time) | Covers 1.35? | Covers 1.36? |
| --- | --- | --- | --- |
| Cilium 1.19.x | 1.32 – 1.35 | Yes | Not yet listed |
| Istio 1.30.x | 1.32 – 1.36 | Yes | Yes |
| Kyverno 1.18.x | 1.33 – 1.35 | Yes | Not yet listed |

Picking 1.36 today would put Cilium and Kyverno outside their documented, e2e-tested Kubernetes range. This will be re-evaluated once Cilium and Kyverno publish 1.36 support (see [`DEPENDENCIES.md`](DEPENDENCIES.md)).

## Items flagged as uncertain compatibility (do not treat as resolved)

1. **Kubernetes 1.36 vs. 1.35** — 1.35 was chosen deliberately for tool compatibility; re-evaluate once Cilium 1.19.x/1.20.x and Kyverno 1.18.x/1.19.x publish 1.36 support.
2. **Helm v4 vs. v3** — v4 is the actively developed line, but third-party chart compatibility across this repo's exact chart set (Cilium, Kyverno, Istio, Prometheus, Grafana, Jaeger, Loki charts) has not been individually verified. Validate per-module in Phases 3–6, fall back to Helm v3.21.3 per-module if needed.
3. **OpenTelemetry Operator exact patch tag** — research surfaced two different numbers (`v0.150.0` and `v0.156.0`) from different sources; re-derive directly from the GitHub releases page before Phase 5 installation.
4. **Ubuntu 24.04 vs. 26.04** — 24.04 chosen for ecosystem tooling maturity at research time; revisit as 26.04 becomes the broadly-documented default across the Kubernetes/Cilium/Vagrant ecosystem.
5. **Istio ambient mode maturity** — upstream now presents ambient mode as production-ready alongside sidecar mode; this repo still starts with sidecar for pedagogical sequencing (ADR-005), not due to a stability concern with ambient.
6. **Kyverno Helm chart's exact values.yaml schema** — `kyverno/install/values-*.yaml` were written against the chart's `main`-branch values structure (top-level keys and per-controller structure confirmed directly), not a byte-for-byte fetch of the `3.8.2`-tagged chart specifically (the exact release tag's raw file wasn't reachable during research). The `replicas`/`resources`/`podDisruptionBudget`/`podAntiAffinity` keys used are stable, long-standing chart conventions unlikely to have changed in a patch-adjacent version, but `helm lint`/`helm template` against the actual pinned `3.8.2` chart (part of `kyverno/tests/static-validation.sh`, run whenever a host has network access to the chart repo) is the real confirmation — treat as unverified until that passes.
